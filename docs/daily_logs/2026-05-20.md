## 新增通路设计

### Text Semantic 通路

- 用户 query：历史 item 的 sentence-transformer 文本 embedding（384-dim）的时间衰减加权均值，与 TwoTower 的 decay_rate=0.8 保持一致
- 所有 item text embedding 预先 L2 归一化（0-text items 保持零向量）
- 无 text history 的用户（本次实验：0 人）返回空候选
- seen-item mask：train+valid（与 v1 一致）

### Popularity Fallback 通路

- 全局按 train 交互数降序排列 item
- 每个用户过滤已见 item，返回 top-K 未见过的全局热门 item
- 无需历史信息，纯全局信号

## 关键结论

### 为什么 4ch_rrf_k60 有效，而 quota 无效？

| 方法 | 机制 | 结果 |
| --- | --- | --- |
| 3/4 路 quota | 名额有限（总 50），加入弱通路 = 从强通路抢名额 | 退步 |
| 3 路 RRF | Text 信号弱，拉低 ICF/TT 的排序信号 | 略低于 v1 |
| **4 路 RRF** | **Pop 独占命中多（2.78%），RRF 信号叠加有效** | **+12.4%** |

### Popularity 通路为何有效

- TwoTower – Pop Jaccard = 0.003，几乎完全独立
- Pop unique hits = 13,816（2.78%）：这些用户的下一个交互是全局热门但个人未见的 item
- 协同过滤（ICF）和学习式模型（TwoTower）对"首次接触热门 item"的用户建模不足，Pop 通路填补了这个缺口
- RRF 的 k 值控制了 Pop 通路的权重：k=60 时 Pop 贡献适当，不会过度主导

### Text Semantic 通路诊断

- 单路 Recall@50 = 2.79%（远低于 ICF 8.36% 和 TT 7.83%）
- Unique hits = 4,788（0.96%）：有小但实在的独立贡献
- Text – Pop Jaccard = 0.0004：与 pop 完全独立
- 在 4ch RRF 中，text 贡献少量排序信号，不显著影响最终结果（4ch 的增益主要来自 pop）
- **Text 通路本身的弱点**：用户 query 基于 item-id embedding 的文本投影，不是 end-to-end 语义匹配；text 覆盖率仅 61.7%；冷物品完全无 text 信号
