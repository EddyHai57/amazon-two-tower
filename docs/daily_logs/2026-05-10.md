## 明日 ID-only Two-Tower debug checklist

明天 ID-only Two-Tower baseline 调试时，至少检查以下内容：

- 数据读取是否正确。
- train/valid/test split 是否一致。
- seen item 是否正确过滤。
- cold item 是否正确 `exclude_from_metric`。
- batch negative 是否合理。
- embedding 是否 L2 normalize。
- temperature 是否设置合理。
- loss 曲线形态是否健康。

loss 曲线判断标准：

1. 正常情况：
   - 前 1-2 个 epoch 的 train loss 应明显下降。
   - 后续 loss 缓慢下降或趋于稳定。
   - valid `Recall@50` / `NDCG@50` 应整体上升或平稳上升。

2. 异常情况 A：
   - loss 很快接近 0，但 `Recall@50` 很低。
   - 可能原因：数据泄露、负样本过于简单、训练目标和评估目标不一致。

3. 异常情况 B：
   - loss 不下降或剧烈震荡。
   - 可能原因：learning rate 过高、batch size 不合适、temperature 不合理、数据读取问题。

4. 异常情况 C：
   - loss 下降，但 valid `Recall@50` 下降。
   - 可能原因：过拟合、评估流程错误、seen item 过滤错误、训练/评估口径不一致。
