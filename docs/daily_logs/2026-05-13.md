## 用户塔 mean pooling 轻量升级

### 本次目标

- 只做用户历史 item embedding mean pooling 的轻量 Two-Tower 升级。
- 不做 Transformer、attention pooling、SASRec、LogQ、负采样大实验、hard negative mining、hybrid retrieval、text-enhanced、item text embedding 或 Faiss 新实验。
- 不覆盖 ID-only baseline checkpoint 或 outputs。

### 新增文件

```text
scripts/train_mean_pool_two_tower.py
configs/two_tower_movies_tv_5core_mean_pool_1epoch.yaml
```

### 模型口径

- item tower：仍为 `item_idx` embedding。
- user tower：`user_id embedding + history_weight * mean(history item embeddings)`。
- `history_max_len=50`，按 `timestamp` 取每个用户最近的 train history item。
- 训练时 history 来自 train split，并对每个训练样本排除当前正样本 item，避免直接把 target item 放进用户历史均值。
- valid 时 history 来自 train split。
- 本轮不运行 full test；后续如运行 test，应使用 train + valid history。

## Hard Negative Mining smoke（Text + Mean Pooling τ=0.15）

### 重要声明

**这是 smoke test，不是最终模型。**
- 不替换当前主模型（Text + Mean Pooling τ=0.15，full test Recall@50=0.076337）。
- Hard Negative Mining 用于探索相似 item 之间的细粒度区分能力，不是用来解决 true new-user cold start。
- 只跑 1epoch limited valid eval，不做 full valid/test、Faiss、20epoch。

### 本次目标

- 验证在 Text + Mean Pooling τ=0.15 架构上加入 Hard Negative Mining 训练链路是否能跑通。
- 观察 1epoch limited eval 是否有继续实验价值。

### Hard Negative 构造方式

- 使用已有 `outputs/item_text_embeddings/movies_tv_5core/item_text_embedding.npy`。
- 训练开始前一次性预计算 HN table：对 153977 个 item，分块（chunk_size=512）在 GPU 上计算 text cosine similarity，每个 item 保存 top-5 最相似 item（排除自身）。
- HN table 构建耗时约 5 秒（GPU 加速）。
- HN table 大小：`(153977, 5)` int32 ≈ 3MB。
- 训练时每个 positive item 从其 top-5 candidates 中选取 2 个：
  - 排除 positive item 自身。
  - 排除该 user 在 train history 中已交互过的 item（降低 false negative 风险）。
- 99.9%（4315539/4319438）的训练样本成功匹配到至少 2 个有效 hard negatives；只有 0.1% 的样本因 history 重叠而跳过。

## Model-based Hard Negative Mining smoke（Text + Mean Pooling τ=0.15）

（2026-05-14 补录）

### 重要声明

**这是 smoke test，不是最终模型。**
- 不替换当前主模型（Text + Mean Pooling τ=0.15，full test Recall@50=0.076337）。
- 只跑 1epoch limited valid eval，不做 full valid/test、Faiss、5epoch 或 20epoch。
- 1epoch limited eval 结果低于 baseline，本轮不建议继续 5epoch。

### 本次目标

- 验证 model-based hard negative mining 链路是否可行。
- 使用当前最终模型 checkpoint 导出的 item embeddings 来选 hard negatives，比 frozen text embedding 更接近"模型真正容易混淆的 negatives"。
- 覆盖全量 153977 个 item，包括 38.3% 没有 text metadata 的 item（text-based HNM 覆盖质量存疑的部分）。

### Hard Negative 构造方式

- 加载当前最终模型：`outputs/text_mean_pool_tau015_20ep/checkpoints/best_model.pt`（epoch=20）。
- 用该模型导出所有 153977 个 item 的 L2 normalized item embeddings（dim=64）。
- 使用 **Faiss IndexFlatIP**（inner product = cosine similarity for L2 normalized vectors）搜索每个 item 的 top-50 nearest neighbors。
- 排除自身后保存为 HN table：`(153977, 50)` int32，**100% 有效条目**（不存在 text-based HNM 中 38.3% 无语义 item 的质量问题）。
- HN table 构建耗时：68.91 秒。
- 训练时每个正样本从其 top-50 candidates 中选取 2 个有效 hard negatives：
  - 排除 positive item 自身。
  - 排除该 user 在 train history 中已交互过的 item。
- **100%（4319438/4319438）的训练样本成功匹配到至少 2 个有效 hard negatives**（top_k=50 大幅降低 history 重叠导致的 false negative 风险）。
